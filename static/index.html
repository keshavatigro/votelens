About VoteLens & how your data loads Show details

VoteLens turns structured election results (who ran, how many votes they received, and where those votes came from) into clear summaries you can scan quickly: turnout, vote shares, close races, and simple monitoring signals—without replacing official canvass or certification tools.

What we expect. Data arrives as JSON describing one election: one or more jurisdictions (counties, wards, precinct groupings), each with contests (offices or questions) and candidates with vote totals. Optional registered_voters and ballots_cast let us compute turnout. Everything stays in your browser session unless you upload a file to this app’s server for processing.

How loading works.

  • First load. When you open this page, VoteLens asks the server for the catalog of sample files and loads the default dataset so charts and tables populate immediately.
  • Sample datasets (below). Click a card to pull that JSON from the server and re-run the insight engine for that election. Use Download JSON on a card to save that exact file locally.
  • Load JSON. Choose a file from your computer that matches the same schema; it is sent to the API, validated, and the dashboard refreshes with your data—nothing is stored unless your deployment is configured to log uploads.
  • Blank JSON template. Downloads an empty, valid starter file you can edit offline, then load back with Load JSON.
  • Reload selected sample. Fetches the current sample again from the server—useful after you change a file under data/ on the machine hosting VoteLens.

Hover KPIs, alerts, table rows, and chart elements for definitions and underlying figures. If the sample grid or downloads fail to respond, confirm the VoteLens server is running and refresh the page.
